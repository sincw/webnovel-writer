# webnovel-init Skill 分析报告

## 一、基本信息

- **Skill 名称**: webnovel-init
- **触发命令**: `/webnovel-init`
- **定位**: 网文项目初始化器（仅 Deep 模式）
- **入口文件**: `.claude/skills/webnovel-init/SKILL.md`

## 二、核心用途

该 Skill 是整个 webnovel-writer 系统的**起点**，负责通过对话式交互收集完整的故事设定信息，然后一次性生成写作就绪的项目骨架。

简单来说：**从零开始，通过问答建立一本新网文的全部基础设定文件。**

## 三、工作流程

```
用户触发 /webnovel-init
    ↓
Wave 1: 收集基础信息（书名/题材/规模/一句话故事）
    ↓
Wave 2: 收集角色信息（主角/感情线/反派分层）
    ↓
Wave 3: 收集金手指设定（类型/代价/成长节奏）
    ↓
Wave 4: 收集世界观 + 创意约束（势力/力量体系/反套路规则）
    ↓
充分性检查（所有必填项是否齐全）
    ↓
运行 init_project.py 生成项目目录和文件
    ↓
写入 idea_bank.json（创意约束数据）
    ↓
填充 大纲/总纲.md（核心设定字段）
    ↓
最终验证（文件完整性检查）
```

## 四、数据收集维度

Skill 通过 4 轮对话（Wave）收集 5 大类数据：

| 维度 | 关键字段 | 说明 |
|------|---------|------|
| A) 项目规模 | 书名、题材(A+B)、目标字数/章数、一句话故事 | 支持复合题材组合 |
| B) 角色阵营 | 主角姓名/欲望/缺陷、感情线配置、反派分层 | 支持单/多主角结构 |
| C) 金手指 | 类型、名称、风格、可见度、代价 | 7种类型各有条件追问 |
| D) 世界观 | 世界规模、势力、力量体系、境界链、货币 | 适配修仙/都市等不同题材 |
| E) 创意约束 | 反套路规则、硬约束、主角缺陷驱动、反派镜像 | v5.4.2 新增的差异化机制 |

## 五、参考文件体系

采用分级加载策略（L0-L3），避免一次性加载过多上下文：

| 级别 | 时机 | 文件 |
|------|------|------|
| L1 | Wave 1 前 | `genre-tropes.md` — 题材套路库 |
| L2 | 题材选定后 | 对应题材模板 + 金手指模板 + 势力系统 + 创意约束文件 |
| L3 | 用户明确请求 | `market-trends-2026.md` — 市场趋势（时效性数据） |

## 六、创意约束系统（核心亮点）

这是 v5.4.2 的重点新增功能，流程为：

1. 根据题材加载反套路库（修仙类 / 规则怪谈类）
2. 生成 2-3 个创意包，每个包含：卖点、反套路规则、主角缺陷、反派镜像、开篇钩子、硬约束
3. 三问筛选：验证题材独特性、人设不可替代性、卖点清晰度
4. 五维评分（满分50）：新颖度 25% | 市场性 20% | 可写性 20% | 爽点密度 20% | 长线潜力 15%
5. 用户选择最终方案

## 七、生成产物

运行完成后，项目目录下会生成：

```
{项目名}/
├── .webnovel/
│   ├── state.json          # 项目状态（书名/题材/目标字数等）
│   └── idea_bank.json      # 创意银行（选中的创意包 + 约束继承）
├── 设定集/
│   ├── 世界观.md
│   ├── 力量体系.md
│   ├── 主角卡.md
│   └── 金手指设计.md
└── 大纲/
    └── 总纲.md             # 填充：一句话故事/核心主线/创意约束/反派分层
```

## 八、安全机制

- **充分性检查**: 6 项必填数据全部就绪才允许运行 init 脚本
- **Hard Fail 条件**: 关键文件缺失或总纲关键字段为空时强制停止
- **回滚/恢复**: 按缺失类型最小化重跑（缺文件→重跑脚本，缺字段→只补字段）
- **目录安全**: 禁止在 `.claude/` 下生成项目

## 九、与其他 Skill 的关系

```
webnovel-init（本 Skill）
    ↓ 生成项目骨架
webnovel-plan（大纲规划，继承创意约束）
    ↓ 生成卷纲/章纲
webnovel-write（章节写作）
    ↓ 产出正文
webnovel-review（质量审查）
```

`webnovel-init` 是整条流水线的第一环，后续所有 Skill 都依赖它生成的 state.json、设定集和总纲。
